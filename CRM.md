# **Цифровая трансформация клиентского опыта для федеральной сети DDX Fitness**

## **Продукт**

**CRM DDX Fitness** — комплексная система управления клиентским опытом для сети фитнес-клубов, объединяющая три ключевые воронки:
- **Продажи** (привлечение новых клиентов)
- **Пролонгация** (удержание и продление подписок)
- **Сервис** (обслуживание и решение проблем)

- 50,000+ клиентов
- 500+ операторов и менеджеров
- 3 различных воронки продаж
- 100+ уникальных сценариев использования
- Интеграция с 8 внешними системами

## **Проблема**

Ключевая цель продукта - развитие внутренней платформы коммуникаций, которая обеспечивает стабильную и эффективную работу контактных центров.
Фитнес-индустрия России сталкивается с:
1. **Высокий отток клиентов** (до 40% ежегодно)
2. **Низкая эффективность контакт-центра** (конверсия звонков 15%)
3. **Разрозненные системы управления** (Bitrix24 + самописные решения)
4. **Отсутствие персонализированного подхода** к клиентам
5. **Сложность масштабирования процессов** при росте сети

## **Процессы**

Как **ведущий продуктовый UX-дизайнер**, я руководил полным циклом редизайна CRM:

### **1. Стратегия и исследования (3 месяца)**
- Провел 25 глубинных интервью с операторами, менеджерами и руководителями
- Организовал 5-day гембу (gemba walk) в контакт-центре в Ярославле
- Проанализировал 1000+ задач в Jira для выявления паттернов использования
- Создал CJM для 3 основных ролей пользователей
- Разработал стратегию AI-first подхода для автоматизации рутинных операций

### **2. Архитектура и проектирование (4 месяца)**
- **Спроектировал универсальную карточку клиента**, адаптирующуюся под:
  - 3 типа пользователей (оператор, менеджер, руководитель)
  - 3 воронки (продажи, пролонгация, сервис)
  - 2 типа данных (клиенты с подписками, контакты без подписок)
  
- **Разработал систему гибких фильтров**, позволяющих:
  - Сохранять состояния фильтрации
  - Каскадную загрузку данных
  - Множественный выбор параметров
  - Контекстно-зависимую логику отображения

- **Создал AI-интерфейсы** для:
  - Автоматического портрета клиента
  - Рекомендаций по коммуникации
  - Анализа успешных/неуспешных сценариев продаж
  - Прогнозирования оттока

### **Дизайн-система**
В течение двух месяцев провел рефакторинг ДС и смерджил с 4 рабочими проектами, попутно решая проблемы синхронизации и актуализации макетов.

- **Создал масштабируемую дизайн-систему** с 150+ компонентами
- Реализовал **адаптивную цветовую семантику** для статусов задач
- Разработал **компоненты массовых операций** для руководителей
- Спроектировал **интеграцию с телефонией** (звонки, SMS, статусы операторов)

## **Решения**

### **Rule-based интерфейс**
- **Роль пользователя** - оператор видит тактические данные, руководитель — стратегические
- дирекция - продажи/пролонгация/[сервис](https://www.figma.com/board/cY218lyIESBQobpPePJLFJ/CRM-Journey-Mapping?node-id=80-4044&t=C7BvMpqAQOmAoTdQ-1)
- **Этап воронки** - определяет доступные действия и приоритеты

### **Дата-виз**
- **Интеллектуальные подсказки** на основе AI-анализа истории клиента
- **Прогрессивное раскрытие информации** (от общего к частному)
- **Визуализация метрик** прямо в интерфейсе выполнения задач

### **Оптимизация рабочих процессов**
- **Сокращение кликов** для частых операций (звонок, SMS, создание задачи)
- **Автозаполнение полей** на основе контекста
- **Групповая обработка задач** для руководителей
- **Offline-first подход** для полевых менеджеров

## **Метрики**
- Конверсия звонков в продажу
- Среднее время обработки задачи
- Процент продления подписок
- Скорость ответа на сервисные обращения
- NPS клиентов
- Производительность операторов

**Качественные улучшения**
- Сократили время обучения новых операторов
- Увеличили прозрачность процессов для руководителей
- Снизили количество ручных ошибок
- Улучшили удовлетворенность сотрудников

## **Технологический стек и методы:**

### **UX Research**
- Jobs-to-be-Done фреймворк
- Контекстные интервью и наблюдения
- Аналитический анализ данных (amplitude, mixpanel)
- Картирование пользовательских путей (CJM, User Flow)

### **Проектирование**
- Информационная архитектура (card sorting, tree testing)
- Прототипирование (Figma, Principle)
- Дизайн-система (Atomic Design)
- Usability testing (регулярные сессии с пользователями)

### **Коллаборация**
- Ежедневные стендапы с delivery-командой
- Координация с Product Owners (3 воронки)
- Работа с ML командой
- Сотрудничество с BI-отделом по метрикам и аналитике

### **Гибкость**
Одна система покрывает три разных бизнес-процесса с разной логикой, но единым интерфейсом

### **AI-интеграция**
- AI предсказывает успешность сценария
- Рекомендует следующие действия
- Анализирует тон переговоров (в планах)

### **Бизнес-ориентированность**
Каждое дизайн-решение привязано к бизнес-метрикам:
- Увеличение LTV клиентов
- Снижение CAC
- Рост операционной эффективности
